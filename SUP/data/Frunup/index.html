Data for runup

This page

First version September 2016. Data are supposed to support study on runup of non-breaking solitary waves on a slope.

The basics

The numerical models in use are a
  • BIM (Boundary integral Model) for full potential theory
  • a Lagrangian Boussinesq type model
These models are run through a set of shell code snippets given in
  • /hom/geirkp/lab/Erika/genrun
These snippets are executed by the newrun facility.

The data

Data are given tar files, for example 
Res0.30slope10.tar
where amplitude/depth = 0.30, slope angle = 10.
Tar files are inflated by 
tar -xz < Res0.30slope10.tar
Synchronization is accrording to t0 in 
 ttim0.30slope10.dat
where simulation starts at t0 with zero defines as in article.

All data are dimensionless with d, the depth, as length scale, and time and velocity scaled by means of d and g.

Examples on data

BIM data 

solrupcasea0.3Bclaw10/a0.30/quad6one6/nf7red0.2_10_3f0.5dxb0.5
where
  • solrupcasea0.3 : the last is a case identifier; the amplitude
  • Bclaw10 : a set identifier followed by the angle
  • /a0.30 : here the amplitude comes again
  • /quad6one6/ : something concerning the shoreline description
  • nf7 : the 7 indicates a spatial resolution; higher numbers are finer resolutions
  • red0.2_10_3f0.5dxb0.5 : more numerical parameters
Datafiles are described with the BIM model

Serre data 

boussrupa0.3BclawBlokkho-sing/n2400rf1
where
  • boussrupa0.3Bclaw : case and set identifier
  • Blokkho-sing : the part following Blokk is equation identifier
    • void : A moderately nonlinear version
    • reg : Fully nonlinear, no singularity when contact angle is 90 deg.
    • sing : Fully nonlinear, singular when contact angle is 90 deg.:
    • <..>ho-lin : with ... as lin, reg or sing. First three options
  • /n2400rf1 : 2400 is number of spatial point, 1 is the maximum Courant number
Datafiles are described with the Serre type model
